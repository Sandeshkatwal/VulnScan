# Future Roadmap

## Near Term

- CI workflow for backend tests and dashboard build.
- Docker or dev container setup.
- PDF export for reports.
- Final dashboard screenshots for GitHub.
- Improved smoke and dashboard tests.

## Medium Term

- Plugin architecture for optional modules.
- SBOM input and software inventory enrichment.
- Advanced passive web checks.
- Better CVE feed handling and update workflow.
- Report template editor.
- Visual regression checks for dashboard pages.

## Long Term

- Role-based access and team workflows.
- Secure deployment model.
- Integration with ticketing systems.
- Optional external platform import without storing credentials.
- Stronger authentication and audit logging.
- Multi-user vulnerability management workflows.

## Safety Direction

Future work should preserve the local-first and authorised-testing-only model. Do not add exploitation, brute force, credential attacks, destructive payloads, external platform tokens, or public API exposure by default.
