Safe Usage Disclaimer

Use VulScan only on systems and web applications you own or have explicit permission to assess. This report is intended for authorised defensive security review.

Authenticated SSH audit uses read-only commands and depends on the permissions of the provided account. Results should be reviewed in operational context.

Limitations

This scan uses TCP connect checks against a fixed list of common ports. Optional HTTP auditing sends only a normal GET request to the root path of detected web services. Optional Web DAST crawling sends bounded same-host GET requests only, can passively review response security headers when requested, does not submit forms, does not authenticate, does not crawl external domains by default, and does not test SQL injection or XSS. Optional TLS auditing performs only a normal TLS handshake and certificate inspection against detected HTTPS services. Optional SSH auditing attempts one authenticated login with explicitly provided credentials and runs read-only Linux inspection commands only, including read-only package manager and configuration indicator checks where available. Optional Windows auditing performs safe reachability checks and, when WinRM authentication is explicitly requested, one harmless validation command plus optional read-only host information, Firewall/Defender status, net accounts policy indicator commands, and exact template-defined registry value checks. It does not install, update, or modify packages or files. It does not perform fuzzing, SYN scanning, UDP scanning, stealth scanning, exploitation, brute forcing, credential attacks, password guessing, payload attacks, destructive checks, password policy changes, account changes, registry value changes, broad registry tree queries, registry hive exports, firewall rule changes, Defender setting changes, service restarts, firewall bypassing, cipher probing, protocol downgrade testing, registry enumeration, share enumeration, credential dumping, or CVE validation.
