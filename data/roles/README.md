# Role and Permission Mapping Samples

These files are fake local/demo examples for Version 21.2 Role and Permission Mapping.

- Use safe role labels only.
- Do not store usernames, passwords, session cookies, bearer tokens, Authorization headers, or secret authentication material.
- `test_account_label` is a documentation label only.
- Linked Session Profiles must be redacted summaries only.
- Permission Matrix entries are planning records and do not trigger live requests.
- Manual Validation Required means the assessor must verify behaviour using authorised test accounts and redacted evidence.

Sample files:

- `sample_roles.json`
- `sample_permission_matrix.json`
