# Authenticated Web Assessment Session Profiles

This directory is for redacted Session Profile examples only.

Do not commit real credentials, passwords, session cookies, bearer tokens, API keys, HAR files, Burp exports, or cookie jars. VulScan 21.0 prefers redacted values and uses profiles only to describe Authentication Context, Authenticated Scope, role notes, and Auth-Required Endpoint classification.

Real local profile files should use ignored names such as:

- `*.local.json`
- `*.secret.json`
- `*.private.json`
- `*real*.json`

The sample profile uses placeholders only.
